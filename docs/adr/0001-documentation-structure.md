# 0001-技术文档结构

## 背景

项目原始 `README.md` 仍是 Vite 模板说明，缺少项目定位、启动方式、核心模块、接口、AI 工作流和部署说明。新成员接手时需要直接阅读源码才能理解项目。

## 决策

在 `docs/` 下建立按主题拆分的技术文档：

- `00-overview.md`：项目总览。
- `01-quickstart.md`：本地启动与开发环境。
- `02-architecture.md`：系统架构。
- `03-business-flow.md`：核心业务流程。
- `04-modules.md`：模块说明。
- `05-data-model.md`：数据模型。
- `06-api.md`：接口说明。
- `07-ai-workflow.md`：AI 工作流。
- `08-deployment.md`：部署与发布。
- `09-troubleshooting.md`：常见问题与排障。
- `adr/`：记录后续重要技术决策。

根目录 `README.md` 作为项目入口，只保留项目简介、快速命令和文档索引。

## 原因

- 按主题拆分比单个长文档更容易维护。
- 编号顺序能引导新人从总览、启动、架构逐步进入细节。
- ADR 可以记录“为什么这么做”，避免技术决策只留在口头或提交历史里。
- 排障文档把已知环境缺口和常见失败点前置，降低接手成本。

## 影响

- 后续修改接口、环境变量、路由、核心状态、AI 任务链路时，需要同步更新对应文档。
- 当前文档基于前端源码整理，后端协议和数据库结构仍需要后端补充正式资料。
