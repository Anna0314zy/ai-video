## 背景与动机

当前项目的前端接口直接依赖旧服务端地址、旧接口路径、七牛云配置、SSE 流式接口和 STOMP 通知通道，后续更换大模型或生成服务会牵动大量前端代码。需要新增一个统一的 NestJS 服务端接口层，由服务端承接全部业务接口和大模型连接，让前端只依赖稳定的项目级 API。

## 变更内容

- 新增基于 NestJS 的服务端应用，作为前端所有接口的统一对接层。
- 将现有前端接口按业务域重新接入 Nest 后端，包括认证、用户信息、项目、剧本、会话、分镜、图片、视频、音频、资源、上传、下载、七牛云凭证和实时通知。
- 新增配置化大模型接入层，配置项覆盖 provider 类型、base URL、模型名称、API key、超时、重试和 provider 扩展参数。
- 通过 provider 抽象屏蔽不同大模型或生成服务的请求、响应和错误差异，后续更换模型优先通过配置或 adapter 完成。
- 支持 SSE 流式聊天和 WebSocket/STOMP 任务通知在新后端下继续工作。
- 统一服务端返回结构、错误码、鉴权处理和配置校验，减少前端对旧接口细节的依赖。
- 增加接口迁移清单、测试和文档，确保所有前端接口都有新后端实现、明确映射或废弃说明。

## 能力范围

### 新增能力

- `configurable-llm-backend`: 覆盖统一 NestJS 后端、全量前端接口重接、配置化大模型 provider 抽象、稳定 API 契约、配置校验、错误归一化、SSE 流式输出、STOMP 实时通知和 provider 切换行为。

### 修改能力

无。

## 影响范围

- 在当前项目或工作区中新增 NestJS 后端应用。
- 新增后端环境变量、配置校验、接口 DTO、服务模块、provider adapter、测试和文档。
- 前端 `apps/web/src/api/models/*`、SSE 请求、下载直链、七牛云凭证请求、WebSocket/STOMP 配置需要重新指向新的 Nest 后端。
- 旧接口路径可以在迁移阶段由 Nest 后端做路径兼容或统一映射，但能力必须由新后端承接，不能代理到旧服务端。
- 服务端将持有大模型和第三方服务密钥，前端不再暴露 provider 细节或敏感配置。
